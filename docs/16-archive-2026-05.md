## 작업 범위 한눈에

이 PR 시점에 한꺼번에 들어간 작업 5개:

## PR 시점 결정·근거 (보존)

### 결정 1: 타임아웃 버퍼 30분
- 운동별 `expected_duration_minutes` + 30분 추가가 "느린 네트워크 vs 장시간 응답 대기" 트레이드오프의 합의점
- 시나리오 3 (느린 네트워크 경계 케이스) 통과 가능

### 결정 2: 동시 갱신 시 AI 결과 우선
- 사용자 운동 데이터 유실 방지 최우선 원칙
- 스케줄러는 충돌 시 양보, AI 콜백은 3회까지 재시도

### 결정 3: TTS는 device TTS, 서버는 설정만
- 서버 음성 합성 비용·지연 회피
- 한국어 단일 정책([`project-korean-only`](../../C:/Users/khjae/.claude/projects/E--init/memory/project_korean_only.md))과 정합

### 결정 4: 피드백 멘트 DB 마스터
- 운영자가 코드 배포 없이 멘트 수정 가능
- 클라이언트는 세션 시작 시 한 번만 fetch

### 결정 5: 발화 로그는 세션 종료 시 배치
- 실시간 REST 호출 비용 회피
- AI가 발화 이벤트 메모리에 누적 후 `POST /internal/feedback/batch` 한 번만

### 결정 6: 임계값 변경은 신규 세션부터 적용
- 진행 중 세션은 영향 없음 — 사용자가 운동 중 임계값이 바뀌면 혼란
- 변경 즉시 적용은 다음 세션부터 자연스럽게
