# 유튜브 영상 → 좌표 확보 (스쿼트 폼 평가 로직 튜닝용)

작성일: 2026-06-05
상태: 분석/추천 (결정 전)
관련: [`2026-05-27-channel-and-youtube-review.md`](./2026-05-27-channel-and-youtube-review.md)

## 0. 목적 한정

이 문서는 **"스쿼트 폼 평가 로직(임계값·기준 시퀀스) 튜닝용 좌표 확보"** 한정이다.
포즈 추정 *모델 학습용* 데이터 수집은 범위 밖 (그건 MediaPipe distillation이라 무의미 — 별도 결론).

## 1. 결론

**현실적으로 가능하고, 추천한다.** 이유는 단순하다 — 좌표 추출 파이프라인이 **이미 구현돼 있다.**

- `app/core/video_processor.py :: analyze_video(path, "squat")` 가 영상 파일 → 프레임별 33개 랜드마크 + 관절 각도 + rep 분석까지 한 번에 뽑는다.
- `app/core/reference_builder.py :: build_reference_sequence(path)` 가 가이드 영상 → rep 분할 → 30프레임 리샘플 → 대표 기준 시퀀스까지 만든다.

즉 유튜브 영상을 `yt-dlp` 로 받아 mp4 로 넘기기만 하면 **기존 코드가 그대로 좌표를 뱉는다.** 신규 추출 코드는 거의 필요 없다.

```
yt-dlp <url> -f mp4  →  analyze_video(path,"squat")  →  frames[].landmarks / frames[].angles
                                                      →  build_reference_sequence(path) → 기준 시퀀스
```

## 2. 무엇을 튜닝하려는가 → 무슨 데이터가 필요한가

현재 평가 로직에 박혀 있는 매직 넘버들이 전부 "데모 한 명 + 고정 프레이밍" 기준으로 손튜닝돼 있다. 외부 영상으로 분포를 확인해 보정하는 게 이 작업의 본질.

| 튜닝 대상 | 위치 | 현재값 | 좌표로 확인할 것 |
|---|---|---|---|
| bottom/standing 무릎 임계 | `squat_analyzer.py` | 100 / 150 | 실제 스쿼트들의 deepest knee 분포 |
| 깊이 부족 경고 | `squat_analyzer.py` | knee > 115 | "good form" 영상의 최저각이 이 선 아래로 잘 내려오나 |
| 과도한 상체 숙임 경고 | `squat_analyzer.py` | torso_tilt > 35 | 측면 영상 torso_tilt 분포 |
| visibility 하한 | `squat_analyzer.py` / `pose_filter.py` | 0.55 / 0.65 | 외부 영상 추적 안정성 |
| 페르소나별 sync 임계 | `constants.py SYNC_THRESHOLDS` | 0.50~0.85 | 다양한 폼 vs 기준 시퀀스의 sync_rate 분포 |
| **대표 기준 시퀀스** | `reference_builder.py` | 단일 가이드영상 | **여러 정자세 영상 평균 → 더 견고한 reference** |

가장 가치 있는 산출물은 마지막 행 — 지금 기준 시퀀스가 **영상 1개에서만** 만들어져 과적합 위험이 큰데, 잘 고른 정자세 영상 N개를 평균하면 DTW 기준선이 훨씬 단단해진다.

## 3. 현실적 장애물 (이 코드 특유)

1. **데모용으로 좁게 튜닝된 필터가 외부 영상을 다 걷어낸다.**
   - `pose_filter.py` 의 `DEFAULT_SQUAT_ROI`(x 0.38~0.78), `max_width=0.34`, `max_aspect_ratio=0.9` 는 *우리 데모 프레이밍* 전용. 유튜브는 피사체 위치·크기가 제각각이라 대부분 프레임이 reject 된다.
   - 다행히 ROI는 `settings.SQUAT_ROI_*` 로 빠져 있음 → **수집용은 ROI 넓히고/끄고, width 상한 완화한 별도 프로파일**로 돌려야 함. (운영 설정은 건드리지 말 것)

2. **2D 투영 + 카메라 각도가 전부를 좌우한다.**
   - 각도는 scale-invariant라 영상 간 비교 OK. 하지만 **측면/정면이 섞이면 같은 자세도 값이 전혀 다름.**
   - 깊이·상체각 = **측면 영상**, 무릎 모음(valgus) = **정면 영상**. → 수집 시 **카메라 각도로 분리 큐레이션 필수.** 현재 `EXERCISE_ANGLES` 는 좌우 평균이라 측면(한쪽 다리 가림)에서 한계 있음.

3. **라벨이 없다.** 임계값 튜닝엔 "good/bad" 약한 라벨이 있어야 검증됨. → 수집 시 영상 단위로 `good_form / shallow / forward_lean / valgus` 정도만 수동 태깅 (영상당 몇 초).

4. **라이선스.** YouTube ToS는 다운로드 금지. 발표/제출물에 "유튜브 수집" 박는 건 리스크. → CC-BY 영상 필터 또는 자체 촬영 우선, 유튜브는 보조. (4-2와 연결)

## 4. 자원 추정

- **수집 규모:** 튜닝 목적이면 대량 불필요. **카메라 각도별 잘 고른 20~40개 클립**(정자세 + 결함 유형별 몇 개씩)이면 임계값 분포 + 기준 시퀀스 다양화에 충분. 큐레이션 > 물량.
- **추출 비용:** MediaPipe CPU 실시간급. 40개 클립 × 수십 초 = **노트북 1~2시간**. GPU 불필요.
- **신규 코드:** `yt-dlp` 래퍼 + 수집용 설정 프로파일 + 결과 좌표를 json 으로 덤프하는 스크립트 (`scripts/` 에 이미 `generate_reference_squat.py`, `annotate_squat_video.py` 있음 → 재사용/확장). 대략 반나절.

## 5. 추천 (결정은 사용자 몫)

1. **수집 소스:** ① 자체 촬영(측면/정면 고정) 우선 + ② 유튜브 CC 영상 보조. (순수 유튜브 대량 수집은 라이선스상 비추)
2. **수집 전용 설정 프로파일** 추가 (ROI off, width 상한 완화) — 운영 설정과 분리.
3. **1차 산출물 2개:**
   - 임계값 분포 리포트 (deepest knee / torso_tilt 히스토그램) → 매직넘버 보정 근거
   - 정자세 영상 N개 평균 기준 시퀀스 → 단일영상 reference 대체
4. ai-server 변경은 메모리 방침대로 **최소화** — 수집은 `scripts/` 오프라인 도구로, 런타임 평가 경로는 건드리지 않음.
